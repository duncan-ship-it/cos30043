Enhancement 1 - Re-formatting for narrow/mobile viewports

Description

When the viewport width of the webpage is under/is shrunken to under 900px, the page's content is transformed appropriately to better adapt to the narrow environment, such as on a mobile phone. This involves the removal of the side navigation, stretching the main content across the entire width of the device, which enables maximum readability of the page's content. Furthermore, the top navigation flows downwards, allowing enough width for each button such that the navigation buttons are still readable and usable, which would not be possible if maintaining the regular horizontal layout.

How it goes above basic requirements

The CSS technique used to implement this enhancement was not covered in any of the lectures or labs so far in this unit. In addition, it is not listed as a requirement for the assignment, but rather extends from the basic requirement of a responsive webpage layout, effectively re-designing portions of the page on demand to suit a client's mobile viewport.

CSS used for enhancement

In order override the CSS from the style.css file at the appropriate time, the browser content is targeted via @media screen and the condition for whether the max width is 900px is tested. When this condition is met (i.e. the viewport width drops below 900px), the CSS selectors embedded underneath are executed, which override many of the styles seen in the style.css file, and re-format the webpage.

Enhancement 2 - Logo animation on cursor hover

Description

Whenever the user hovers on the logo featured in the top left of all webpages on the site, an animation is played, which gives the impression the car is driving off. Not only does this boost the interactivity the client has with the webpage, making it more exciting to the user, but also introduces brand personality, with the hyperbolic movements of the car leaving the user to infer a friendly and jovial brand.

How it goes above basic requirements

While in a lecture the implementation of CSS animation was mentioned, the actual code behind this enhancement needed to be researched, hence again going above the material taught thus far. Additionally, the assignment did not require the use of animations, but I chose to use them to improve the cosmetics of the site, as well as making navigating the site a more engaging experience for the user.

CSS used for enhancement

First the animation needed to be defined. This is done through the @keyframes keyword, which defines the transformations undertaken on each frame, as well as the when the frames occur in relation to the duration of the animation. In order to target the object (in this case the logo image), the attribute "animation-name" is utilized underneath the image selector, which links the animation to this object. Lastly, the attribute "animation-duration" was used to specify the duration of the entire animation, such that it lasts long enough to be noticeable (in this case 3 seconds).
